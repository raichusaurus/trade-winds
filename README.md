# trade-winds

Fresh implementation of Trade Winds.

This project is the pilot project for using [Loom](https://github.com/raichusaurus/loom) as a workflow framework. The previous implementation has been preserved as `trade-winds-legacy`; this repository starts from a clean foundation.

## Loom Workflow

Project decisions and phase outputs live in [`docs/loom`](docs/loom):

- [Progress Ledger](docs/loom/PROGRESS.md)
- [01 Discovery](docs/loom/01-discovery.md)
- [02 Requirements](docs/loom/02-requirements.md)
- [03 Architecture](docs/loom/03-architecture.md)
- [04 Planning](docs/loom/04-planning.md)
- [05 Contracts, Tests, CI/CD](docs/loom/05-contracts-tests-cicd.md)
- [06 Implementation](docs/loom/06-implementation.md)
- [07 Retrospective](docs/loom/07-retrospective.md)

The first pass starts with Discovery before any application scaffolding.

Update the progress ledger whenever changes are pushed to `main`.

The root progress ledger stays concise and links to child ledgers for detailed contract and implementation progress.
